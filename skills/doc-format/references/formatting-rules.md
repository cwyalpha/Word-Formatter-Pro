# Formatting Rules

## 标题层级

| 类型 | 识别规则 | 默认字体 | 默认字号 |
|---|---|---|---:|
| 题目 | 文档开头连续居中且字体字号一致的段落；TXT/MD 首个非层级标题段落 | 方正小标宋简体 | 22 |
| 副标题 | 题目下方居中且字体字号不同的段落 | 楷体_GB2312 | 16 |
| 一级标题 | `一、`、`二、` 等 | 黑体 | 16 |
| 二级标题 | `（一）`、`（二）` 等 | 楷体_GB2312 | 16 |
| 三级标题 | `1.`、`2.`、`1．` 等 | 仿宋_GB2312 | 16 |
| 四级标题 | `（1）`、`(1)` 等 | 仿宋_GB2312 | 16 |
| 正文 | 其他普通段落 | 仿宋_GB2312 | 16 |

二级标题与正文在同一段落且以句号分隔时，会尝试在段内拆分标题部分和正文部分，分别应用二级标题和正文字体。

## 页面和段落

- 页边距：上 3.7cm / 下 3.5cm / 左 2.8cm / 右 2.6cm。
- 页脚距：2.5cm。
- 正文行距：28 磅。
- 题目/副标题行距：33 磅。
- 页码：默认奇偶分页，奇数页右对齐，偶数页左对齐。
- 页码字体：宋体，14pt。
- 正文和标题会清理残留缩进，再设置标准首行缩进 2 字符。
- `force_a4=false` 时 Word/WPS 文档保留原纸张大小；TXT/MD 默认使用 A4。

段落包含图片、嵌入对象等特殊内容时，只处理必要文字格式，保留段落行距等布局，避免图片或对象显示不全。

## TXT/MD 特殊处理

- `.txt` 会按文本行生成段落。
- `.md` 会清理 Markdown 标记，包括标题 `#`、粗体/斜体、链接、图片语法、引用、分隔线等。
- TXT/MD 中手写数字编号（如 `1.`、`1.2.3`）按源文档保留，不自动递增重排。
- 默认空行模式为：删除单个空行，连续多个空行合并为 1 个空行。
- 可通过 `--blank-line-mode` 或配置字段 `blank_line_mode` 改为空行不处理，或保留单个空行但压缩多个空行。

空行模式：

| 模式 | 行为 |
|---|---|
| `不改动任何空行` | TXT/MD 原始空行保持不变 |
| `删除单个空行，多个空行保留至1个空行` | 单个空行删除，连续多个空行合并为 1 个 |
| `保留单个空行，多个空行保留至1个空行` | 单个空行保留，连续多个空行合并为 1 个 |

## 表格、图表和附件

- 表格内容自动调整默认关闭，避免改变复杂表格布局。
- 启用后可设置表头字体、表格字体、字号、行距、行高、表格宽度、边框、列宽和智能对齐。
- 表格智能对齐启用后，会尝试将表头、合计/总计、序号列、短文本居中，将数字内容右对齐。
- 图/表标题通过图片或表格上下相邻的居中 `图...` / `表...` 段落识别。
- 附件标识默认启用格式化，识别 `附件`、`附件1`、`附件：` 等形式，并会段前分页。
- 附件标识后会继续尝试识别附件题目和副标题；如果没有附件题目/副标题，不应跳过紧随其后的正文段落。

## 符号和特殊内容

- 符号标准化默认关闭，可通过配置 `normalize_punctuation: true` 启用。
- 启用后会较保守地处理中文语境中的英文标点、引号、括号、省略号等，避免影响数字、小数、英文内容。
- 段落含域代码时会跳过符号标准化，避免破坏自动域、书签、批注引用等特殊内容。
- 删除段首空白 run 时会检查 run 内是否含有非文本 OOXML 子元素，避免破坏域代码、书签、批注引用等特殊内容。

## 平台和自动编号

- Windows 环境下，会尽量通过 WPS/Word COM 接受修订，并将自动编号转为普通文本。
- `.doc/.wps` 转 `.docx` 在 Windows 优先使用 WPS/Word COM；macOS/Linux 或 COM 转换失败时，会尝试调用 LibreOffice `soffice`。
- 如果 `soffice` 自动查找失败，可在 CLI 中使用 `--soffice <路径>` 指定；如果转换仍失败，应先手动另存为 `.docx` 后再处理。
- 非 Windows 或 COM 不可用时，会跳过需要 WPS/Word 的预处理步骤。
- 自动编号转换失败或被跳过时，输出文件中编号样式可能仍由 Word/WPS 自动编号控制，字体字号和缩进需要人工检查。
